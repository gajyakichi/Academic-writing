## 🎯 Introduction の構成ルール

### **黄金の三段落構成：Known → Unknown → やったこと**

Introduction は必ず以下の 3 つのパートで構成します。

### **第一段落：Known（知られていること）**

**目的**: 研究分野の基本情報と既存の知識を提供する

**含めるべき内容**:

- 研究対象の定義
- 研究分野の重要性・背景
- 既存研究で明らかになっている事実
- 基本的な疫学データや統計
- 関連する病態生理や機序（既知のもの）

**執筆のポイント**:

- 一般的な内容から具体的な内容へ（漏斗型）
- 広い視点から始めて、徐々に研究テーマに絞り込む
- 必ず文献引用で裏付ける

**例**:

```
メタボリックシンドローム（MetS）は、中心性肥満、高血圧、脂質異常症、
インスリン抵抗性を含む一連の病態として定義され、心血管疾患（CVD）や
死亡率のリスク増大との関連から、世界的に公衆衛生上の重大な問題として
浮上している[1,2]。日本においても、MetSの有病率はここ数十年、生活様式や
食習慣の変化と並行して着実に増加している[3,4]。特定健診制度により、
40-74歳を対象としたスクリーニングが全国的に実施されており...
```

---

### **第二段落：Unknown（知られていないこと）**

**目的**: 既存研究のギャップや未解決の問題を明確にする

**含めるべき内容**:

- 既存研究の限界
- 未検討の領域・疑問
- 矛盾する研究結果
- 本研究の必要性の根拠

**執筆のポイント**:

- "However"や"Nevertheless"などの逆接で始めることが多い
- 具体的に何が不明なのかを明示する
- なぜその問題が重要なのかを説明する
- 本研究がそのギャップを埋めることを示唆する

**よく使う表現**:

- しかしながら、〜は不明である
- 〜については十分に検討されていない
- 先行研究では〜が限定的であった
- 〜の関連性は明らかになっていない
- 特に〜については解明されていない

**例**:

```
しかしながら、MetSの有病率と心臓突然死（Sudden Cardiac Death, SCD）の
発生率との直接的な関連性は十分に検討されていない。特に、地域ごとの
MetS有病率の差がSCD発生率にどのように影響するかは不明である。さらに、
MetSの前段階であるPre-MetSとSCDの関連や、致死的な心室性不整脈
（shockable rhythm）と非致死的リズム（non-shockable rhythm）を
分けた解析は行われていない。
```

---

### **第三段落：やったこと（本研究の目的）**

**目的**: 本研究が何を明らかにするかを簡潔に述べる

**含めるべき内容**:

- 研究の目的（objectives/aims）
- 研究デザインの概要
- 使用データや対象の簡潔な説明
- 主要な解析手法（詳細は Methods で）
- 期待される成果や意義（オプション）

**執筆のポイント**:

- 明確で具体的な目的を述べる
- Unknown で提示した問題に対する解決策として提示
- 簡潔に（詳細は Methods と Results で）
- 研究の独自性や新規性を強調

**よく使う表現**:

- 本研究では、〜を目的とした
- 本研究の目的は〜を明らかにすることである
- 〜を検討するため、本研究を実施した
- 我々は〜を用いて〜を解析した

**例**:

```
本研究では、日本の全都道府県を対象に、特定健診データを活用し、MetSおよび
Pre-MetSの有病率とSCDの発生率との相関を検討する。特に、shockable rhythm
（心室細動や心室頻拍）とnon-shockable rhythm（無脈性電気活動や心静止）
それぞれについて分析を行い、地域差がSCDリスクに及ぼす影響を明らかにする
ことを目的とする。
```

## 📑 各セクションの構成

### **Abstract（要旨）**

**構成** (150-250 語):

1. **背景** (1-2 文): 研究の重要性
2. **目的** (1 文): 本研究の目的
3. **方法** (2-3 文): 研究デザイン、対象、解析方法
4. **結果** (2-3 文): 主要な発見（数値を含む）
5. **結論** (1-2 文): 研究の意義と示唆

**ポイント**:

- 過去形で書く（結果は過去形）
- 略語は初出時に定義
- 具体的な数値を含める

---

### **Methods（方法）**

**構成**:

1. **研究デザイン**: 観察研究、前向き、後ろ向きなど
2. **対象**: 包含基準、除外基準、サンプルサイズ
3. **データ収集**: 使用したデータベース、測定方法
4. **定義**: 主要な変数や診断基準の定義
5. **統計解析**: 使用した統計手法、有意水準

**ポイント**:

- 再現可能なように詳細に記述
- 時系列に沿って記述
- 過去形で書く
- 倫理承認について記載

---

### **Results（結果）**

**構成**:

1. **対象の特性**: ベースライン特性、患者背景
2. **主要な結果**: 主要評価項目の結果
3. **副次的な結果**: その他の重要な発見
4. **サブグループ解析**: 必要に応じて

**ポイント**:

- 過去形で書く
- 数値は必ず具体的に（信頼区間、p 値を含む）
- 図表を効果的に使用
- 図表の内容を文章で繰り返さない（補完する）
- 解釈や考察は含めない（Discussion で）

---

### **Discussion（考察）**

**推奨される構成**:

1. **主要な知見の要約** (1 段落)

   - 本研究の主要な発見を簡潔に要約
   - "We found that..."で始めることが多い

2. **先行研究との比較** (2-3 段落)

   - 結果が一致する先行研究
   - 結果が異なる先行研究とその理由
   - 本研究の新規性

3. **メカニズムの考察** (1-2 段落)

   - 結果の生物学的・病態生理学的説明
   - 仮説の提示

4. **臨床的意義** (1 段落)

   - 結果の実臨床への応用
   - 政策や公衆衛生への示唆

5. **Limitations（研究の限界）** (1 段落)

   - 研究デザインの限界
   - データの限界
   - バイアスの可能性

6. **結論** (1 段落)
   - 主要な発見の再強調
   - 今後の研究への示唆

**ポイント**:

- 過度な推測を避ける
- Limitations は正直に書く
- 因果関係の主張には注意（観察研究の場合）

---

### **Conclusion（結論）**

**構成** (1 段落、3-5 文):

1. 主要な発見の要約
2. 研究の意義
3. 今後の展望（オプション）

**ポイント**:

- 新しい情報は含めない
- 簡潔に
- 研究の重要性を強調

## ✍️ 文章作成のベストプラクティス

### **明確で簡潔な文章**

1. **1 文 1 メッセージ**

   - 複雑な文を避ける
   - 長い文は分割する

2. **能動態を使う**

   - ❌ 解析が実施された → ✅ 我々は解析を実施した
   - ただし、Methods では受動態も可

3. **具体的な数値を使う**

   - ❌ 多くの患者 → ✅ 43,565 人の患者
   - ❌ 有意な差 → ✅ 有意な差（p < 0.001）

4. **略語の使用**
   - 初出時に正式名称と略語を記載
   - 例: メタボリックシンドローム（MetS）
   - その後は略語を使用

### **論理的な流れ**

1. **段落ごとに 1 つのトピック**
2. **トピックセンテンス**（各段落の最初の文で主題を示す）
3. **接続詞の効果的使用**
   - Therefore, However, Moreover, In addition, など

### **時制の使い分け**

| セクション                 | 時制   | 例                          |
| -------------------------- | ------ | --------------------------- |
| Introduction（既知の事実） | 現在形 | MetS is defined as...       |
| Introduction（先行研究）   | 過去形 | Previous studies showed...  |
| Methods                    | 過去形 | We analyzed data from...    |
| Results                    | 過去形 | The prevalence was 15.2%... |
| Discussion（本研究の結果） | 過去形 | We found that...            |
| Discussion（一般的事実）   | 現在形 | These findings suggest...   |

## 📚 参考文献の管理

### **BibTeX 形式での管理**

`references.bib`ファイルに以下の形式で記載:

```bibtex
@article{citation_key,
  author    = {著者名},
  title     = {論文タイトル},
  journal   = {ジャーナル名},
  year      = {出版年},
  volume    = {巻},
  pages     = {ページ}
}
```

### **引用のルール**

1. **引用キーの命名規則**

   - `第一著者の姓` + `年` + `キーワード`
   - 例: `alberti2009harmonizing`

2. **文中での引用**

   ```latex
   MetSは世界的な問題である\cite{alberti2009harmonizing}。
   複数の研究で報告されている\cite{alberti2009harmonizing,grundy2005diagnosis}。
   ```

3. **引用のタイミング**
   - 事実の主張には必ず引用
   - 数値やデータには必ず引用
   - 一般的に知られている事実は引用不要

## ✅ チェックリスト

### **投稿前の最終チェック**

#### **構造**

- [ ] Introduction は Known → Unknown → やったこと の順序になっている
- [ ] Abstract は背景・目的・方法・結果・結論を含む
- [ ] 各セクションが論理的に繋がっている

#### **内容**

- [ ] 研究の目的が明確
- [ ] 方法が再現可能なレベルで詳細
- [ ] 結果に具体的な数値が含まれる（信頼区間、p 値）
- [ ] Discussion で主要な発見を要約している
- [ ] Limitations を正直に記載している

#### **文章**

- [ ] 略語は初出時に定義している
- [ ] 時制が適切に使い分けられている
- [ ] 各段落に 1 つのトピック
- [ ] 過度に長い文がない

#### **引用**

- [ ] すべての事実・データに引用がある
- [ ] 引用形式が統一されている
- [ ] references.bib に全ての文献が登録されている

#### **図表**

- [ ] すべての図表にキャプションがある
- [ ] 本文中で全ての図表に言及している
- [ ] 図表は自己完結的（それだけで理解できる）

#### **技術的**

- [ ] LaTeX がエラーなくコンパイルできる
- [ ] PDF の見た目が適切
- [ ] 参考文献リストが正しく生成されている

## 💡 よくある質問

### Q: Introduction はどれくらいの長さが適切？

A: 通常、1-2 ページ（約 3-5 段落）。ジャーナルのガイドラインを確認してください。

### Q: Discussion で新しい結果を示しても良い？

A: いいえ。すべての結果は Results セクションで示すべきです。Discussion では解釈のみ。

### Q: Limitations はどれくらい詳しく書くべき？

A: 重要な限界は全て記載すべきですが、些末な点は避けます。研究の信頼性を損なわない範囲で正直に。

### Q: 自分の先行研究を引用しても良い？

A: はい、関連する場合は適切です。ただし、過度な自己引用は避けましょう。
